Proximus' prepaid cards are implemented on the Alcatel Open Services Platform. The platform is a service oriented architecture and runs on a cluster of Tru64 alpha nodes.

New capabilities were added to allow more innovative pricing schemes. I designed and executed the acceptance tests. I used Mercury TestDirector for

  • gathering test requirements,
  • designing tests,
  • drawing up a test plan,
  • scheduling and guiding test execution,
  • managing defects.

Although the test campaigns were focused on functional increments, they contributed to an overall improvement of the service quality by adopting a more rigorous test approach than had hitherto been the case and by improving the delivery and defect follow-up procedures.

I wrote a Ruby script to check the accounting tickets for anomalies. The script has proved useful both to improve test productivity and as an early warning system in the production environment.

I developed an end-to-end automated test chain based on FIT tables. Initially I used the FIT implementation to drive the test fixtures, but the FIT framework was subsequently replaced by a proprietary implementation of a FIT-like framework using WinRunner scripts in order to smoothly integrate call tests with TestDirector.
